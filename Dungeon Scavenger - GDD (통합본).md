## 1. 게임 개요

| 항목 | 내용 |
| :--- | :--- |
| **게임명** | Dungeon Scavenger |
| **장르** | 2D 탑다운 루팅 / 탈출형 어드벤처 |
| **플랫폼** | PC (브라우저 기반 HTML5, 추후 독립 실행형 확장) |
| **플레이 모드** | 싱글플레이어 (추후 멀티플레이어 확장 고려) |
| **코어 타겟** | 전략적 판단과 긴장감을 즐기는 캐주얼~미드코어 게이머, 로그라이트 및 생존 어드벤처 팬 |
| **전투 비중** | 낮음 — 탐색, 루팅, 회피, 탈출이 핵심 |

### 한 줄 정의

**"이미 공략이 끝난 폐던전에 홀로 들어가, 남겨진 잡동사니와 잊혀진 유물을 챙기고 살아서 같은 길로 빠져나오는 저전투 루팅 탈출 게임."**

### 핵심 재미 3가지

1. **가치 판단의 재미** — 무게·부피·가격·희귀도·위험 대비 가치를 비교하며 *무엇을 담고, 무엇을 버릴지* 결정하는 순간들.
2. **버티다 빠져나오는 긴장감** — 체력, 조명, 인벤토리 포화, 짐 무게로 인한 소음 증가가 누적되며 *지금 나가야 하나, 조금만 더 뒤져야 하나*의 딜레마.
3. **폐허 수색꾼의 몰입감** — 영웅의 무용담이 아닌, 깨진 상자와 버려진 배낭 사이에서 가치를 건져내는 **조심스럽고 찝찝한** 탐험의 감각.

## 2. 핵심 게임플레이 루프

### 메인 루프

```
거점 준비 → 던전 진입 → 탐색 및 루팅 → 위험 조우 → 탈출(단일 출구) → 정산 및 성장 → 다음 탐험 준비
```

### 세부 루프 구조

| 단계 | 플레이어 행동 | 핵심 긴장 요소 |
| :--- | :--- | :--- |
| **거점 준비** | 장비 구매, 가방 정리, 탐험 목표 설정, 지도 열람 | 자금 배분 결정 |
| **진입 및 탐색** | 방·복도 이동, 오브젝트 조사, 경로 선택 | 시야 제한, 미지 구역 |
| **루팅 및 판단** | 아이템 획득·감정·선별, 인벤토리 재조정 | 무게 누적 → 소음 증가 |
| **위협 조우** | 도주·은신·우회·최소 전투 | 몬스터 어그로, 함정 |
| **탈출** | *왔던 길*로 복귀, 입구까지 안전 귀환 | 과적 상태의 느린 이동, 재배치된 위험 |
| **정산 및 성장** | 아이템 판매·감정, 장비 업그레이드 | 수익 극대화 판단 |

### 루프가 재미를 만드는 원리

- **긴장감**: 던전 체류 시간이 길어질수록, 짐이 무거워질수록 위험이 누적된다.
- **보상감**: 살아서 탈출하면 그 순간까지의 모든 판단이 수익으로 전환된다.
- **중독성**: 큰 전투 없이도 *더 갈까, 지금 돌아갈까*의 선택이 매 판 반복적으로 쾌감을 만든다.

## 3. 세계관 및 스토리라인

### 세계관

용사와 모험가 파티들에 의해 **완전히 공략**되어 그 기능을 잃어버린 판타지 세계의 '폐던전'. 오랫동안 방치된 이 공간에는 영웅들이 미처 챙기지 못한 잔해들이 남아있다.

- 마왕은 이미 쓰러졌다. 던전의 핵은 파괴되었다.
- 경비 골렘은 대부분 기능을 멈추었지만, **완전히 해제되지 않은 소수**가 여전히 배회한다.
- 함정 기계 장치는 동력을 잃었지만, **일부는 여전히 작동**한다.
- 보물은 이미 대부분 털렸지만, **영웅들의 눈에 띄지 않은 잡동사니와 잊혀진 유물**이 곳곳에 남아있다.

### 플레이어 캐릭터: 스캐빈저(Scavenger)

플레이어는 영웅이 아니다. **뒤늦게 남은 것을 건지러 온 수색꾼**이다. 전투 능력보다 기민함과 판단력이 생존의 핵심이다.

- 강한 용사들이 던전을 클리어한다는 소문이 돌면, 뒤를 따르는 생업형 수색꾼들이 있다.
- 이들에게 폐던전은 위험한 폐허이자, **일확천금의 기회**다.
- 죽은 모험가의 배낭, 버려진 마도 부품, 오래된 귀족 장식품 — 이 모든 것이 시장에서 돈이 된다.

### 스토리라인 및 세계 확장

단일 던전에서 시작하되, 반복 탐험과 유물 수집을 통해 **폐던전의 역사가 조금씩 드러나는** 레이어드 내러티브를 지향한다.

- **탐험 일지 & 메모**: 던전 안에서 발견되는 오래된 기록물들이 세계관 조각을 채워간다.
- **유물 파편 세트**: 동일 세트의 유물 파편을 모으면 던전의 과거 사건이 해금된다.
- **거점 NPC의 대사 변화**: 수집한 유물과 기록물에 따라 NPC가 세계관 힌트를 제공한다.
- **숨겨진 구역 해금**: 특정 유물 조각을 모으면 지도에 표시되지 않던 비밀 구역이 열린다.

> 스토리는 플레이어에게 **강요되지 않는다**. 관심 있는 플레이어만 기록물을 챙겨 읽을 수 있도록 설계한다.

## 4. 주요 시스템 설계

### 4-1. 소음 & 어그로 시스템 ★ (핵심 독창 시스템)

짐의 무게와 캐릭터 상태가 **소음 수치**에 직접 영향을 미친다. 이 시스템이 루팅의 위험도를 자연스럽게 상승시킨다.

| 상태 | 소음 수준 | 어그로 반응 반경 |
| :--- | :---: | :---: |
| 가벼운 상태 (인벤토리 50% 이하) | 낮음 | 좁음 |
| 중간 상태 (인벤토리 50~80%) | 중간 | 보통 |
| 과적 상태 (인벤토리 80~100%) | 높음 | 넓음 |
| 초과 상태 (중량 한계 초과) | 매우 높음 | 전체 구역 |

- 과적 상태에서는 이동 시 **둔탁한 발소리**가 발생하며 화면 가장자리에 피로도 이펙트가 표시된다.
- 몬스터는 소음 반응형 AI를 가진다 — 직접 시야에 들어오지 않아도 소리를 듣고 접근한다.
- 소음을 유발하는 다른 요소: 상자 뚜껑 여닫기, 달리기, 무거운 물건 던지기, 문 부수기.

### 4-2. 단일 출구 & 왕복 동선 시스템 ★

- **입구 = 유일한 출구**. 들어왔던 길로만 나갈 수 있다.
- 가벼운 몸으로 진입했다가, 루팅 후 **짐을 잔뜩 짊어지고 같은 위험 구역을 역주행**해야 한다.
- 이 구조는 별도의 장치 없이도 극적인 귀환 텐션을 만든다.
- 심화 레벨에서는 이미 지나온 구역에 **새로운 위험이 재배치**되거나 구조물이 변화한다.

### 4-3. 인벤토리 시스템

**그리드 + 무게 혼합형** 인벤토리를 사용한다.

- 가방에는 **칸 수(슬롯) 제한**과 **무게 제한**이 동시에 존재한다.
- 아이템마다 차지하는 슬롯 크기가 다르다 (1×1, 1×2, 2×2 등).
- 무게가 누적될수록 이동 속도 감소 + 소음 수치 증가 (4-1 시스템과 연동).

**선택이 발생하는 순간 설계:**
- 큰 유물을 담으면 다른 아이템 공간이 막힌다.
- 귀환 중 더 좋은 아이템 발견 시 기존 아이템을 버려야 한다.
- 회복약 공간을 유지할지, 수집품을 한 개 더 담을지 결정해야 한다.

**탐험 현황 UI (상시 표시):**

| 항목 | 표시 내용 |
| :--- | :--- |
| 예상 수익 | 현재 인벤토리 기준 예상 판매가 |
| 위험 상태 | 안전 / 주의 / 위험 / 위급 |
| 소음 수준 | 아이콘 + 색상으로 시각화 |
| 귀환 거리 | 입구까지의 대략적 거리 |

### 4-4. 긴장감 설계 시스템

**함정**
- 바닥 압력판, 늦게 발동하는 화살 함정, 무너지는 바닥, 독가스 분출, 소리 유발 함정 (몬스터 유인).
- 반드시 **전조를 보여준다** (바닥 균열, 벽 구멍, 색 다른 타일). 처음엔 피할 수 있어야 공정하며, 숙련 시 빠르게 통과하는 숙련 플레이가 가능해야 한다.

**소수 몬스터 (개성 중심)**

| 몬스터 | 특성 | 회피 전략 |
| :--- | :--- | :--- |
| 수호 해골 | 느리지만 시야가 넓음 | 시야 밖 우회 |
| 박쥐 떼 | 소리에 반응 | 정숙 이동 |
| 순찰 슬라임 | 특정 구역 배회 | 타이밍 파악 |
| 잔존 경비 골렘 | 한번 발각 시 집요하게 추적 | 최우선 도주 |

- 각 몬스터는 **싸우는 법**보다 **피하는 법**이 먼저 떠오르도록 디자인한다.

**자원 압박**
- 체력, 조명 지속 시간, 붕대/회복약, 함정 해제 도구, 투척 유인 아이템.
- 자원은 넉넉하지 않지만 박하지 않게 — 깊이 들어갈수록 부족해지는 체감이 핵심.

**귀환 압박**
- 강제 타이머 대신 **약한 압박의 누적**: 조명 소진 근접, 일부 구역 구조물 붕괴, 적 배회 경로 변화.

### 4-5. 아이템 시스템

**아이템 분류표**

| 등급 | 예시 | 특징 |
| :--- | :--- | :--- |
| **일반 잡템** | 깨진 촛대, 녹슨 단검, 빈 병, 동전 주머니 | 자주 발견, 낮은 판매가, 초반 경제 기반 |
| **실용 자원** | 밧줄, 횃불, 붕대, 기름병, 약초 | 판매보다 사용 가치가 높아 고민 유발 |
| **고가 수집품** | 은잔, 보석 세공품, 귀족 문양 상자 | 돈벌이 핵심, 무게/부피가 애매하게 높음 |
| **희귀 유물** | 봉인 문양 조각, 고대 지도 조각, 탐험 일지 | 판매가 + 메타 성장 + 스토리 해금 가치 |

**잡템에도 가치를 부여하는 방식:**
- **세트 수집 보너스**: 같은 문양의 은식기 3종, 특정 시대 장비 세트를 완성하면 추가 보상.
- **의뢰 납품**: 거점 NPC가 특정 잡템을 반복 요청 (유리병 5개, 마도 파편 3개 등).
- **제작 재료**: 녹슨 금속 → 수리 재료, 낡은 천 조각 → 붕대 제작.

### 4-6. 위험 회피 및 제한적 전투

전투는 **최후의 수단**이자 **짧고 단순**하게 처리된다.

**회피 우선 선택지:**
- 돌아가기 / 기다리기 / 문 닫기 / 적 시야 피하기 / 함정 우회 / 던진 물건으로 소리 유도

**전투 처리 (최소 구현):**
- 근접 1회 타격
- 뒤로 물러서기 / 회피
- 소모형 투척 도구
- 단, 전투는 체력·소모품·시간 손실이 크므로 항상 비효율로 설계한다.

## 5. 던전 구조 설계

### 5-1. 기본 구조: 방(Room) 기반 + 연결 통로

각 방은 **명확한 용도**를 가진다:

```
[입구 구역] - [저장 창고] - [붕괴 복도] - [소형 예배실]
                    |                              |
              [폐숙소]                        [경비실]
                    |                              |
              [보관고]  -------- [하층 진입문] -----+
```

- 방 단위로 루팅 포인트, 적 출현, 함정 배치, 특수 테마를 독립적으로 설계한다.
- **입구 = 출구** 구조를 명확히 하여, 플레이어가 항상 자신의 귀환 경로를 인식하게 한다.

### 5-2. 맵 생성 방식: 반고정 구조 + 랜덤 배치

초기 개발에는 **완전 수제 맵**을 권장하며, 일부 요소만 랜덤화한다:

| 고정 요소 | 랜덤 요소 |
| :--- | :--- |
| 방의 배치와 연결 | 상자 내용물, 아이템 종류 |
| 전체 던전 레이아웃 | 함정 활성 여부 |
| 주요 이벤트 방 | 적 초기 위치 |
| 입구 위치 | 잠긴 문 / 지름길 개방 여부 |

### 5-3. 반복 플레이를 위한 변형 방식

절차 생성 없이도 아래 변형으로 리플레이 가치를 확보한다:
- 동일 방에서 조사 가능한 오브젝트 조합 변경
- 일부 구역이 붕괴 상태로 시작
- 보물방 후보 3곳 중 1곳만 활성화
- 탐험 진입마다 위험도 수치 소폭 변동

### 5-4. 레벨 가독성 원칙

- 좁은 복도 남발 금지
- 방의 용도가 실루엣만 보아도 구분 가능하게
- 인터랙션 포인트는 눈에 띄게
- 위험 타일은 배경과 미세하게 차이 나게

---

## 6. 성장 시스템

### 6-1. 성장 축 3가지

**경제 성장 (돈 → 장비)**

| 장비 | 효과 |
| :--- | :--- |
| 가방 업그레이드 | 인벤토리 슬롯/무게 한계 증가 |
| 랜턴 업그레이드 | 시야 반경 증가, 지속 시간 증가 |
| 장갑/도구 세트 | 함정 해제 성공률 증가 |
| 경량 신발 | 이동 속도 향상, **소음 감소** (소음 시스템 연동) |
| 간이 지도 도구 | 탐색한 방 자동 기록 |
| 귀환 표식 장치 | 긴급 탈출 보조 (사용 횟수 제한) |

**정보 성장 (탐험 → 지식)**
- 특정 구역 지도 해금
- 함정 유형 사전에 기록
- 희귀 아이템 출현 힌트 획득

**탐험 숙련 성장 (반복 → 효율)**
- 조사 속도 향상
- 아이템 감정 비용 할인
- 특정 종류 아이템 발견 확률 소폭 상승

### 6-2. 성장 설계 원칙

> 시스템 성장 40% + 플레이어 숙련도 60%

- 장비가 좋아져도 **무조건 안전하지 않다**.
- 경로 판단, 귀환 타이밍, 인벤토리 관리는 **항상 플레이어의 몫**이다.
- 이 균형이 장기 플레이를 지속적으로 재미있게 만든다.

## 7. 아트워크 및 연출 방향

### 7-1. 그래픽 스타일

**저해상도 픽셀아트 기반 2D 탑다운**

- 제한된 색상 팔레트, 낮은 채도
- 과한 애니메이션 없이 **분위기 우선**
- 실루엣으로 오브젝트 용도가 구분되는 타일셋

**스타일 키워드:** 바랜 색, 낡은 석재, 먼지, 균열, 녹, 부식

### 7-2. UI/UX 방향

**"정보는 많이, 복잡해 보이지 않게"**

필수 HUD 요소:
- 체력 바
- 조명 잔량 (아이콘 + 수치)
- 인벤토리 공간 / 현재 무게
- 소음 수준 (색상 코딩: 초록 → 노랑 → 주황 → 빨강)
- 예상 수익 / 귀환 거리 힌트

UX 세부 원칙:
- 아이템에 커서/포커스 시 간단 정보 즉시 표시
- 가치 높은 아이템은 테두리 또는 아이콘으로 구분
- 과적 상태 즉시 체감 가능 (화면 주변부 피로도 이펙트)
- 버릴 물건 선택 시 예상 손실 수익 표시

### 7-3. 사운드 & 분위기 연출

**전투보다 분위기가 중요한 게임에서 사운드는 핵심 서사 수단이다.**

| 사운드 유형 | 내용 |
| :--- | :--- |
| 발소리 (상태 연동) | 가벼울 때 — 가볍고 조용한 발걸음 / 과적 시 — 둔탁하고 무거운 발소리 |
| 환경음 | 물방울 소리, 먼 거리 낙하음, 바람 소리, 쥐 소리 |
| 오브젝트음 | 오래된 문 삐걱임, 상자 뚜껑 열리는 소리, 금속 마찰음 |
| 위험 전조음 | 함정 작동 전 기계음, 몬스터 존재감 암시음 (울음보다 기척) |
| BGM | 강한 음악 없이 **희미한 드론음 + 환경음 중심** / 위험 시만 저음 드론 증가 |

### 7-4. 폐던전 감성 연출

핵심 감정: **조심스러움, 발견의 기쁨, 욕심과 불안, 살아 돌아온 안도감**

- 빈 방도 의미 있게 만들기 (누군가 먼저 털고 간 흔적)
- 깨진 장비, 버려진 배낭 등 **스토리 오브젝트** 곳곳에 배치
- **탈출 정산 연출 강화**: 오늘의 수익 / 발견물 목록 / 위험도 / 미회수 아이템 등 리뷰 화면

## 8. 기술 스택 및 개발 로드맵

### 8-1. 기술 스택

| 영역 | 기술 | 역할 |
| :--- | :--- | :--- |
| **백엔드 (게임 서버)** | Python + asyncio + websockets | 물리 충돌, 몬스터 AI, 상태 계산, 소음 계산 전담 |
| **프론트엔드 (클라이언트)** | HTML5 Canvas API + Vanilla JS | 렌더링, 입력 처리, UI 표시 |
| **통신 프로토콜** | WebSocket (JSON / MessagePack) | 양방향 실시간 통신 |
| **구조 패턴** | 서버 권위형 (Authoritative Server) | 클라이언트 조작 차단, 상태 정합성 보장 |

> 소음 어그로 반경 계산, 무게에 따른 이동 속도 계산 등 **핵심 시스템은 반드시 서버에서 처리**하여 클라이언트 위변조를 방지한다.

### 8-2. 단계별 개발 로드맵

| 단계 | 목표 | 핵심 구현 항목 |
| :--- | :--- | :--- |
| **Phase 1: 재미 검증** | 핵심 루프 프로토타입 | 이동, 루팅, 인벤토리 압박, 탈출, 정산 |
| **Phase 2: 긴장감 추가** | 위협 요소 도입 | 함정 1~2종, 몬스터 1종, 조명 제한, 소음 시스템 기초 |
| **Phase 3: 시스템 완성** | 핵심 시스템 통합 | 소음 어그로 연동, 단일 출구 귀환 텐션, 아이템 세트 보너스 |
| **Phase 4: 반복성 강화** | 리플레이 가치 확보 | 맵 변형, 메타 성장, 아이템 종류 확대, NPC 의뢰 |
| **Phase 5: 완성도 향상** | 폴리싱 | UI 정제, 사운드, 연출, 저장 구조, 밸런싱 |

### 8-3. MVP 필수 기능 체크리스트

- [ ] 플레이어 이동 (8방향 또는 4방향 + 충돌 판정)
- [ ] 방 기반 맵 (최소 5~8개 방 + 출입문 연결)
- [ ] 오브젝트 조사 및 아이템 획득
- [ ] 인벤토리 (슬롯형 + 무게 기초)
- [ ] 아이템 가치 시스템 (판매가 + 무게)
- [ ] 소음 수치 기초 (무게 연동)
- [ ] 함정 1~2종 + 몬스터 1종
- [ ] 체력 시스템 + 단일 출구 탈출
- [ ] 거점 정산 루프 (판매 → 다음 탐험 시작)
- [ ] 기본 HUD (체력, 조명, 소음, 예상 수익)
