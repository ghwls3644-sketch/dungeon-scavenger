# 2D 탑뷰 스캐빈저 어드벤처 기획 및 개발 가이드

## 1. 게임 핵심 컨셉 및 세계관
* **장르**: 루팅(Looting) 및 탈출형 어드벤처
* **시점 및 모드**: 2D 탑뷰 (Top-Down) 시점의 싱글 플레이어 (추후 멀티 플레이어 확장 가능성 고려)
* **전투 비중**: 낮음 (전투보다는 탐색, 자원 수집, 함정 회피에 초점을 맞춤)
* **시놉시스**: 영웅과 선발 모험가들에 의해 이미 완벽하게 공략되어 기능이 정지된 판타지 배경의 '폐던전'
* **주인공**: 폐던전을 뒤져 남겨진 잡동사니와 잊혀진 유물을 수집하는 스캐빈저(Scavenger)
* **위협 요소**: 던전 핵심 기능은 상실되었으나, 극히 드물게 해제되지 않은 함정이나 살아남은 몬스터가 도사리고 있음

## 2. 핵심 게임 루프
| 단계 | 상세 내용 |
| :--- | :--- |
| **거점 준비** | 던전 진입 전 획득한 재화로 탐험 장비를 구매하고 지도를 보며 동선을 계획함 |
| **진입 및 탐색** | 이미 답파된 던전이라 시중에 지도가 존재하지만, 실제 지형과 다른 변수(무너진 샛길 등)가 존재함 |
| **파밍** | 잡동사니와 유물을 수집하며, 인벤토리 무게 제한을 고려하여 가치가 높은 아이템을 선별함 |
| **위협 조우** | 소음에 반응하는 골렘 등 적이나 함정에 맞서 싸우지 않고 도주나 은신으로 극복함 |
| **귀환 및 탈출** | 들어왔던 입구가 곧 유일한 출구가 되는 단일 출구 시스템을 통해 빠져나와야 함 |

## 3. 핵심 시스템: 스캐빈저의 무게 및 텐션 조율
* **과적 페널티 (이동 속도 저하)**: 전투 능력이 떨어지는 비전투원의 육체적 한계를 반영하여, 무게 한계를 초과할수록 이동 속도가 저하됨
* **소음 발생 및 어그로**: 배낭이 꽉 찰수록 이동 시 부딪히는 소음 파동이 크게 발생하며, 이는 위험 요소를 깨우는 트리거가 됨
* **기회비용 딜레마**: 무거운 고가의 유물을 챙기기 위해 필수 생존 장비를 버려야 하는 고민이 강제됨
* **단일 출구의 부메랑 동선**: 가벼운 상태로 진입했다가 무거워진 몸으로 같은 길을 되돌아 나와야 하는 왕복 구조로 생존 텐션을 극대화함
* **캐릭터 상태 피드백 강화**: 작위적인 환경 변화 없이, 짐이 무거울수록 발소리가 둔탁해지고 시야 가장자리에 피로도 효과를 더해 심리적 압박감을 조성함

## 4. 기술 스택 아키텍처
* **개발 환경**: 상용 게임 엔진(Unity, Unreal 등) 사용을 금지하고 Python 및 HTML5 기반으로 구축함
* **구조 설계**: 게임의 핵심 로직을 서버에서 처리하는 '서버 권위형(Authoritative Server)' 구조로 클라이언트 조작을 차단함
* **백엔드 (게임 서버)**: Python, asyncio, websockets를 사용하여 물리 충돌, 몬스터 AI, 상태 계산 등을 전담함
* **프론트엔드 (클라이언트)**: HTML5 Canvas API와 Vanilla JavaScript를 활용하여 렌더링 및 입력을 전송함
* **통신 프로토콜**: WebSocket (데이터 포맷: JSON 또는 MessagePack)을 통해 양방향 실시간 통신을 진행함

## 5. 단계별 개발 로드맵
| 마일스톤 | 핵심 목표 |
| :--- | :--- |
| **Phase 1: 기반 구축** | Python WebSocket 서버 및 HTML5 Canvas 통신 구축, 플레이어 이동 및 동기화 검증 |
| **Phase 2: 핵심 루프** | 2D 타일맵 시스템, 충돌 처리, 아이템 배치 및 인벤토리 획득 로직 프로토타이핑 |
| **Phase 3: 위협 추가** | 시야 마스킹, 패턴 배회형 몬스터 AI, 접촉 시 작동하는 함정 오브젝트 구현 |
| **Phase 4: 정식 빌드** | 2D 스프라이트 애니메이션 적용, UI/UX 고도화, 사운드 효과 추가 및 최적화 |
